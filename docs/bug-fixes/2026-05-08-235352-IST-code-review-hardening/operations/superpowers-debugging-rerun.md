## Phase 1: Root Cause Investigation

Original findings were rechecked from source:

- Booking status transitions were exposed through the shared server mutation, so UI restrictions alone were insufficient.
- The sort dropdown was visually hidden by opacity, scale, and pointer events, but its option buttons could remain keyboard-visible.
- Puppeteer was present in production dependencies even though it is browser automation tooling.
- A local final test account guide needed an explicit ignore rule to reduce accidental commit risk.
- Generated site-map docs could render raw template expressions such as dynamic `tel:` values.

## Phase 2: Pattern Analysis

Working project patterns used:

- Booking business rules belong in server-side data mutations, not only in React buttons.
- Animated menus can remain mounted, but hidden interactive descendants must be removed from tab traversal.
- Tooling/browser automation dependencies belong in `devDependencies`.
- Generated docs should be regenerated by the generator, not manually edited.
- Local credential or account guides should stay outside tracked source or be ignored explicitly.
